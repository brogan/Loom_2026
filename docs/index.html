How the tools fit together

Step 1 — Draw

Bezier Draw

Java · Swing · bezier/

An interactive vector editor for creating the geometric source material that Loom renders. Draw with the mouse; every shape is a cubic Bézier spline.

  • Draw closed polygon sets — the main input for Loom's subdivision engine
  • Draw open curve sets — non-closed splines rendered as stroked paths or brush stamps
  • Draw point sets — discrete positions for dot or brush-stamp effects
  • Organise geometry into named layers
  • Weld polygons together, cut with the knife, extrude edges
  • Save as XML files consumed directly by the Parameter Editor and Loom

Step 2 — Configure

Parameter Editor

Python · PyQt6 · loom_engine/loom_parameter_editor/

A GUI for configuring every aspect of the Loom rendering pipeline, without editing XML by hand. Organises configuration into tabs matching the rendering pipeline stages.

  • Geometry tab — register polygon sets, open curve sets, and point sets
  • Subdivision tab — define multi-stage recursive subdivision rules (polygon types only)
  • Shapes tab — pair geometry with subdivision parameters
  • Sprites tab — position shapes on canvas, configure animation and morph targets
  • Rendering tab — define stroke, fill, point, and brush renderer styles
  • Run tab — launch, reload, and capture output from Loom
Parameter Editor Help →
Step 3 — Render

Loom Engine

Scala · Swing · loom_engine/

The rendering engine. Reads all XML config files, builds the scene, and draws it frame by frame — either as a static composition or a continuous animation.

  • Recursively subdivides polygon geometry through configurable multi-stage pipelines
  • Applies per-stage point-level transforms (spiking, curving, tearing)
  • Renders sprites with STROKED, FILLED, POINTS, or BRUSHED renderers
  • Animates with random jitter, keyframe easing, or polygon vertex morphing
  • Exports high-resolution stills and animation frame sequences
  • Reloads configuration instantly on F5 without restarting

Typical workflow

  1. In Bezier Draw, draw polygon sets (and optionally open curves or point sets). Save to the project's polygonSets/, curveSets/, or pointSets/ directories.
  2. In the Parameter Editor, open the project. Register the geometry in the Geometry tab. Define subdivision rules (Subdivision tab). Compose shapes (Shapes tab). Configure renderers (Rendering tab). Position sprites and set animation (Sprites tab).
  3. Press Run Loom (Cmd+L). The editor saves all config files and launches the Loom engine.
  4. In the Loom window, press F5 to reload after any parameter change, S to save a still, V to record animation frames.
  5. For the fastest iteration: edit in the Parameter Editor → press Cmd+R (Reload) → Loom updates in under a second without restarting.

File communication

All three applications communicate via files in the project directory — there are no sockets or live connections. Bezier Draw writes geometry XML; the Parameter Editor writes configuration XML; the Loom engine reads them all. Reload and capture commands are signalled via small sentinel files (.reload, .capture_still, .capture_video) that Loom watches for on each draw cycle.
