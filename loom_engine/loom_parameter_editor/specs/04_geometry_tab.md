# Geometry Tab

**Source:** `ui/geometry_tab.py` (container), plus six sub-tab files  
**Model:** `models/polygon_config.py`, `models/open_curve_config.py`, `models/point_config.py`, `models/oval_config.py`  
**IO:** `file_io/polygon_config_io.py`, `file_io/open_curve_config_io.py`, `file_io/point_config_io.py`, `file_io/oval_config_io.py`

## Purpose

The Geometry tab groups all geometry-source configuration into one `QTabWidget` container with six sub-tabs: Spline Polygons, Regular Polygons, Curves, Points, Ovals, and Bitmap Polygons. All sub-tabs share a common signal interface.

## Container — `GeometryTab`

`GeometryTab` owns a single shared `PolygonSetLibrary` instance that is wired into both `SplinePolygonTab` and `RegularPolygonTab`. Changes made in either polygon tab are immediately visible in the other.

### Signals forwarded to `MainWindow`

| Signal | Meaning |
|---|---|
| `modified` | Any geometry change |
| `shapeLibraryChanged` | Shape list changed |
| `subdivisionChanged` | Subdivision assignment changed |
| `spriteLibraryChanged` | Sprite assignment changed |
| `rendererLibraryChanged` | Renderer assignment changed |
| `newShapeCreated(set_name, shape_name)` | New shape was added |
| `newSpriteCreated(set_name, sprite_name)` | Quick Setup created a new sprite |

### Directory setters (called by `MainWindow._notify_tabs_of_project_dir()`)

```
set_polygon_sets_directory(d)
set_regular_polygons_directory(d)
set_curve_sets_directory(d)
set_point_sets_directory(d)
set_oval_sets_directory(d)
set_bitmap_polygon_dirs(polygon_sets_dir, background_image_dir)
```

## Sub-tab 1: Spline Polygons

**Source:** `ui/spline_polygon_tab.py`  
**XML output:** `polygonSets/<name>.poly.xml` (the polygon geometry files themselves) + `configuration/polygons.xml` (the library registry)

### Purpose

Manages a library of named polygon sets. Each polygon set is a reference to an external `.poly.xml` file created by the Bezier editor. The tab lets you add/remove sets from the library, view polygon counts, and assign each set to subdivision params and sprites via a Quick Setup convenience panel.

### Layout

Horizontal splitter — left list, right detail panel.

**Left panel:**
- "Polygon Sets" label
- `QListWidget` showing all registered polygon sets; each item shows name, file, polygon count, and usage count (number of sprites referencing this set)
- Buttons: `+ File` (add existing file), `+ Set` (create empty), `- Set` (remove), `Rename`, `Duplicate`, `↑ Up`, `↓ Down`, `Bake…` (trigger Bake in SubdivisionTab)
- File system `polygonSets/` directory browser

**Right panel (detail):**
- `Name` field
- `File` path (read-only, or browse to link an existing file)
- Polygon count label
- Sprite usage count label
- Quick Setup panel: dropdown selectors for SubdivisionParamsSet and RendererSet, `Create Sprite` button

### Key Behaviour
- `_refresh_file_list()` scans `polygonSets/` directory for `.poly.xml` files and auto-registers any that are not yet in the library.
- `_reconcile_polygon_sets()` cross-references the library against existing files and updates polygon counts.
- Quick Setup: selects a polygon set + subdivision params + renderer set, then calls `SpriteTab._add_sprite()` with those references pre-filled.

## Sub-tab 2: Regular Polygons

**Source:** `ui/regular_polygon_tab.py`  
**Dialog:** `ui/regular_polygon_dialog.py`  
**IO:** `file_io/regular_polygon_io.py`  
**Storage:** `regularPolygons/<name>.xml` (editor-only; not read by engine)

### Purpose

Generates regular n-gon polygon sets (triangle, square, pentagon, …) within the editor itself, without requiring the Bezier editor. The generated polygon is stored as an XML file and registered in the shared `PolygonSetLibrary`.

### Regular Polygon Parameters

| Parameter | Description |
|---|---|
| `sides` | Number of sides (3–100) |
| `radius` | Circumscribed radius |
| `center_x/y` | Centre position |
| `rotation` | Initial rotation angle |
| `name` | Display name |

### Dialog — `RegularPolygonDialog`

A simple form with spin boxes for the above parameters. On accept, writes the polygon as an XML file to `regularPolygons/` and adds the resulting `PolygonSet` entry to the shared library.

## Sub-tab 3: Curves

**Source:** `ui/open_curve_tab.py`  
**Model:** `models/open_curve_config.py`  
**IO:** `file_io/open_curve_config_io.py`  
**XML file:** `configuration/curves.xml`; geometry files in `curveSets/`

### Purpose

Manages a library of open-curve sets. Each curve set references a `.curve.xml` file generated by the Bezier editor.

### Model — `OpenCurveSetLibrary`

```
OpenCurveSetLibrary
  name : str
  curve_sets : List[OpenCurveSet]

OpenCurveSet
  name : str
  file : str      # filename in curveSets/
  enabled : bool
```

### Layout

Similar to Spline Polygons: list on left, detail on right. Directory scanner for `curveSets/` auto-registers new `.curve.xml` files.

## Sub-tab 4: Points

**Source:** `ui/point_tab.py`  
**Model:** `models/point_config.py`  
**IO:** `file_io/point_config_io.py`  
**XML file:** `configuration/points.xml`; geometry files in `pointSets/`

### Purpose

Manages named point sets — ordered lists of 2D coordinates used as vertex positions for sprites. Point sets may be created manually (inline editor) or loaded from files in `pointSets/`.

### Model — `PointSetLibrary`

```
PointSetLibrary
  name : str
  point_sets : List[PointSet]

PointSet
  name : str
  file : str      # filename in pointSets/, or empty for inline
  enabled : bool
  points : List[Point2D]   # used for inline point sets

Point2D
  x : float
  y : float
```

## Sub-tab 6: Bitmap Polygons

**Source:** `ui/bitmap_polygon_tab.py`  
**Dialog:** `ui/widgets/bitmap_polygon_dialog.py`

### Purpose

Generates polygon sets by tracing the outline of a bitmap image. The user specifies a source image and threshold parameters; the tab produces a `.poly.xml` file which is saved to `polygonSets/` and automatically appears in the Spline Polygons list.

### Bitmap Polygon Dialog Parameters

| Parameter | Description |
|---|---|
| Source image | PNG/JPG to trace |
| Threshold | Luminance threshold for edge detection |
| Invert | Invert mask before tracing |
| Smoothing | Number of smoothing passes |
| Min polygon size | Discard polygons smaller than this |
| Output name | Name for the resulting `.poly.xml` file |

On confirm, the tracing algorithm produces a `PolygonSet` XML file and triggers `SplinePolygonTab._refresh_file_list()`.

## Shared Signal Interface (All Sub-tabs)

Every sub-tab exposes the same set of signals. `GeometryTab` forwards all of them:

| Signal | When emitted |
|---|---|
| `modified` | Any data change |
| `shapeLibraryChanged` | Shape list changed |
| `subdivisionChanged` | Subdivision assignment changed |
| `spriteLibraryChanged` | Sprite library changed |
| `rendererLibraryChanged` | Renderer assignment changed |
| `newShapeCreated(str, str)` | New shape added |
| `newSpriteCreated(str, str)` | Quick Setup created a sprite |

## Cross-Tab Dependencies

- `SplinePolygonTab` and `RegularPolygonTab` share one `PolygonSetLibrary` — changes in either are reflected in both.
- When `BitmapPolygonTab.modified` fires, `SplinePolygonTab._refresh_file_list()` is called to pick up the new `.poly.xml`.
- The Sprites tab receives the polygon library, open curve library, point library, and oval library so its geometry-source dropdowns are kept in sync.
