Overview

Loom is a Scala-based drawing engine that generates complex visual patterns through recursive polygonal subdivision. Starting from simple 2D shapes — triangles, squares, or custom bezier curves — Loom subdivides them into hundreds or thousands of smaller polygons, applies geometric transforms at multiple levels, and renders the results as high-resolution stills or animations.

The Loom Parameter Editor provides a graphical interface for configuring all aspects of the engine. Instead of editing XML by hand, you define polygons, subdivision rules, rendering styles, and animation parameters through tabbed panels, then launch Loom directly from the editor.

What Loom Produces

  • Complex geometric tessellations from simple polygon subdivision
  • Organic, distorted forms through point-level transforms (tearing, spiking, curving)
  • High-resolution stills suitable for large-format printing (via the quality multiplier)
  • Animations where shapes rotate, scale, morph, and shift over time
  • Layered compositions with multiple sprites at different positions and rendering styles
  • Brush-stroke textures that stamp along polygon edges, optionally following a meandering organic path rather than the exact Bézier geometry
  • Morph animations that smoothly blend between different polygon shapes
  • Open-curve geometry (non-closed splines) rendered as stroked paths New
  • Discrete point sets (scatter patterns and point-stamp effects) New

Architecture: Configuration Domains

Polygons / Open Curves / Points + Subdivision Params → Shapes
Shapes + Renderers + Position & Animation → Sprites
Sprites + Global Config → Canvas Output
Each domain is stored in its own XML file within the project directory. The Scala engine reads all files at startup (or on reload) and renders accordingly. Note: Subdivision applies only to polygon-type shapes (polygon sets and regular polygons). Open curves and point sets are not subdivided — they are rendered directly.

Getting Started

Prerequisites

  • Java JDK 11+ installed and on your PATH
  • Scala / SBT installed and on your PATH
  • Python 3.10+ with PyQt6 for the parameter editor
  • Bezier Draw (Python) (optional) — a companion PySide6 application for drawing custom polygon sets, open curve sets, and point sets. Located at bezier_py/; launched automatically by the editor via the Geometry tab Create/Edit buttons.

Creating a New Project

  1. Launch the editor. Select File > New Project (Cmd+N).
  2. Enter a project name and choose a parent directory (default: ~/.loom_projects). The default projects directory can be changed in the Global Config tab under Project Directory.
  3. Click OK. The project folder and all sub-directories are created, and all tabs populate with defaults.

Typical Workflow

  1. Configure the canvas (Global tab) — Set width, height, background colour, background image.
  2. Define base geometry (Geometry tab) — Create polygon sets, open curve sets, or point sets. Use the Polygons sub-tab for spline or regular polygons; the Curves sub-tab for open non-closed splines; the Points sub-tab for discrete point collections.
  3. Set up subdivision (Subdivision tab) — Create a Params Set with one or more stages. Each stage = one recursion level. 3–5 stages typically yields good complexity. Subdivision applies to polygon types only.
  4. Compose shapes (Shapes tab) — Choose a source type (Polygon Set, Open Curve Set, or Point Set) and pair it with a subdivision params set (polygon types only).
  5. Define renderers (Rendering tab) — Set stroke, fill, colour, and dynamic change behaviour. For brush textures, choose the BRUSHED mode and configure the brush library.
  6. Create sprites (Sprites tab) — Position shapes on the canvas, set size, assign a renderer set, and configure animation.
  7. Configure drawing settings (Run tab) — Set quality multiple, animating toggle, draw background once.
  8. Run Loom (Cmd+L) — The engine launches and renders. Use Cmd+R to reload after parameter changes.
  9. Capture output — Use the Run tab's Save Still / Save Animation buttons.
Start with a single square polygon, 2–3 stages of QUAD subdivision, and a basic stroked renderer. This gives immediate visible output. Add transforms and complexity incrementally.

High-Resolution Stills

For print-quality output, set Quality Multiple (Run tab) to 4–12. This multiplies the canvas pixel dimensions. When the quality multiple is greater than 1, the on-screen display shows a scaled-down preview of the full high-resolution canvas; the saved PNG captures the full resolution.

At high quality multiples, stroke widths do not automatically scale unless Scale Image is enabled. Always enable Scale Image when rendering for print.

Animations

  1. Enable Animating (Run tab).
  2. Configure sprite animation (Sprites tab → Animation panel).
  3. Choose Draw Background Once behaviour: enabled = trail/layering effect; disabled = clean each frame.
  4. Click Save Animation to start recording frames to renders/animations/.

Bezier Draw Application

Bezier Draw is a companion PySide6 (Python) application for creating the polygon sets, open curve sets, and point sets that Loom uses as base geometry. You do not draw shapes by entering numbers — you draw them visually on an interactive canvas, then save them to the project's geometry directories for Loom to load.

The Python version (bezier_py/) adds two new capabilities over the original Java version: Mesh Build Mode for constructing polygon meshes from traced point sets, and a persistent Trace Image layer that is saved with the document.

The Drawing Canvas

The canvas has a 1000×1000 pixel drawing area with a 20px edge offset on all sides (so the physical canvas is 1040×1040 internally, then scaled to fill the window). Polygon coordinates are saved in a normalised form (centred on 0, typically in the range −0.5 to +0.5) — the pixel positions are converted automatically on save. You draw at canvas-pixel resolution; Loom works with normalised coordinates.

A grid of lines is overlaid on the canvas. Toggle its visibility with the Toggle Grid toolbar button.

Selection Modes

Three selection modes let you select and move geometry at different levels of granularity. All three support two sub-modes:

  • Relational (default): movements propagate through weld links — welded neighbours move together to maintain continuity.
  • Discrete: only the directly selected item moves; welded neighbours stay put.

Press Shift while a selection mode is active to toggle between Relational and Discrete. Cmd+click on the toolbar button when switching to a mode also activates Discrete directly.

Layer System

Bezier Draw supports multiple named layers. Each polygon is assigned to a specific layer. The layer system has two main uses: organisational separation while drawing, and multi-layer export for use as a Polygon Set Bundle in Loom.

Layer behaviour while drawing

  • The active layer (bold name in the Layers table) receives all new geometry. Every click in Create Polygons or Create Points mode adds to the active layer.
  • The active layer is drawn at full opacity. Other visible layers are drawn at 20% opacity (ghosted) so you can trace or reference them without confusion. Hidden layers (Vis unchecked) are not drawn at all.
  • Selection and hit-testing only operates within the active layer — you cannot accidentally select or move polygons on other layers.

Multi-layer export

When multiple layers contain polygons, clicking Save produces a separate XML file per layer (named {overallName}_{layerName}.xml) plus a {overallName}.layers.xml manifest file. The Loom Parameter Editor detects the manifest and offers to import the whole bundle as a layered polygon set reference.

Use layers to separate logically distinct parts of a complex polygon arrangement — for example, inner shapes on one layer and outer border shapes on another. Each layer can then be assigned different rendering styles in Loom.

Mesh Build Mode New

Mesh Build Mode allows you to construct a closed polygon mesh by sequencing existing source points. You hover over points to add them to a face sequence, then press P to commit the polygon. All committed faces accumulate in an auto-created Mesh layer; adjacent faces are welded at shared vertices automatically.

Step-by-Step: Building a Mesh from Traced Points

  1. Load a trace image — use File → Load Trace Image… to load a reference photograph or drawing. It appears as a semi-transparent Trace layer.
  2. Create source points — switch to Create Points Mode (h) and click to place anchor dots at the key vertex positions on your trace (corners, intersections, outline features). Save when done.
  3. Activate Mesh Build Mode — click the Mesh toolbar button or press Ctrl+A. The button highlights in orange. A Mesh layer is created automatically.
  4. Sequence a face — move the mouse near source points. Any point within 10 px is auto-added to the sequence and shown with a green numbered ring. Points between 10 and 18 px show a larger preview ring but are not added yet.
  5. Commit the face — press P when you have sequenced all vertices of one polygon face (minimum 3). The polygon is added to the Mesh layer; vertices coincident with existing Mesh layer anchors (within 5 px) are welded automatically.
  6. Continue — the sequence clears; mode stays active. Build the next adjacent face. Repeat until the mesh is complete.
  7. Save — click Save. The Mesh layer writes to polygonSets/; the point layer writes to pointSets/.
Suggested settings: Work with quads (4-vertex faces) for clean subdivision in Loom. Use Weld All Adjacent after building to catch any near-coincident seams. Keep control point handles visible while building to confirm weld quality.
Source points are not removed or consumed when used in a face. Each point can be used in multiple adjacent faces — this is intentional and necessary for shared-vertex mesh topology.

Spline Polygons

Loom works primarily with cubic bezier spline polygons. Each side of a polygon is a bezier curve defined by four points:

  • Anchor points (start and end of the curve) — the actual vertices of the polygon
  • Control points (two per side) — invisible handles that shape the curve

A square has 4 sides and therefore 16 points total (4 × 4). An N-sided polygon has 4N points. This structure matters because subdivision and transforms operate on these specific point types separately.

Open Curves

Open curve sets are collections of non-closed cubic Bézier splines — paths that have a distinct start and end rather than looping back. They are drawn in Bezier Draw using the "Finish Open Curve" button instead of the normal "Close Polygon" button. Open curves are not subdivided; Loom renders them directly as stroked paths or brush stamps. They cannot be used as morph targets.

Point Sets

Point sets are collections of discrete anchor points drawn in Bezier Draw. Each point can be rendered as a dot stamp (POINTS mode) or a brush stamp (BRUSHED mode). Point sets are not subdivided and do not support morph targets.

Recursive Subdivision

Each stage in a Params Set takes every polygon and divides it into sub-polygons. The process repeats: stage 2 subdivides the output of stage 1, and so on.

  • A square with 3 stages of QUAD subdivision → 43 = 64 polygons (4 per side, each stage)
  • A square with 5 stages of QUAD → 1024 polygons
  • A triangle with 3 stages of TRI → 33 = 27 polygons
  • SPLIT modes produce 2 polygons per stage: 3 SPLIT stages → 23 = 8 polygons

Each stage can use a different subdivision type with its own parameters. You might use QUAD for stage 1, then SPLIT_VERT for stage 2, then apply spiking transforms in stage 3.

Transforms at Multiple Levels

Loom applies transforms at three distinct levels:

  1. Inset transform — Scales/rotates the entire subdivision result relative to the parent polygon centre, creating gaps or overlaps between levels.
  2. Whole polygon transforms (PTW) — Randomly translate, scale, or rotate selected sub-polygons as units, controlled by probability.
  3. Point-level transforms — Manipulate specific subsets of points (exterior anchors, central anchors, control points) within each sub-polygon. These create organic distortion effects like spiking, tearing, and curving.

Shapes vs Sprites

Shapes are abstract blueprints: a polygon set + subdivision rules. The same shape can be used by multiple sprites.

Sprites are concrete instances: a shape placed at a specific position on the canvas, with a size, rotation, animation behaviour, and visual renderer. You can have many sprites sharing one shape, each appearing differently.

Animation Modes

Sprites support three mutually exclusive animation approaches:

  • Random (Jitter) — Each frame applies random perturbations to scale, rotation, and position within configurable ranges. Creates organic, continuous motion. Available for all geometry types.
  • Keyframe — Interpolates between explicitly defined keyframe states (position, scale, rotation) using one of 41 easing algorithms. Creates precise, controllable animation. Available for all geometry types.
  • Morph — Blends the polygon vertices between the base shape and a morph target shape. Comes in jitter-morph (random amount each frame) and keyframe-morph (eased over keyframes) variants. Available for polygon sets and regular polygons only. Not available for open curves or point sets.

Pressure-Sensitive Open Curves

Open curves drawn with a pressure-sensitive stylus and tablet carry per-anchor pressure values (0.0–1.0) that are saved in the openCurveSet XML. In Loom's BRUSHED renderer, these values modulate the brush stamp size and/or opacity along the stroke — lighter stylus pressure produces smaller/more transparent stamps, heavier pressure produces larger/more opaque ones.

Complete Workflow

  1. Draw with a stylus. In the Curves sub-tab click Create in Bezier. In Bezier Draw, select a pressure-sensitive drawing tool (requires a Wacom or similar tablet). Draw open curves — click to place anchor points while varying stylus pressure. The anchor dot sizes in Point Selection mode show recorded pressure levels.
  2. Save from Bezier Draw. Click Save in Bezier Draw. The open curves are written to curveSets/{name}.xml with pressure="0.xxx" attributes on each anchor point. Close Bezier Draw — the Curves sub-tab auto-detects the new file and selects it.
  3. Register the curve set. In the Curves sub-tab, ensure the file is selected in the dropdown. Note the curve set's Name — you will reference this from the Shapes tab.
  4. Create a Shape and Sprite. In the Shapes tab, create a new Shape and set its source to Open Curve Set, selecting your curve set by name. In the Sprites tab, create a Sprite referencing that Shape.
  5. Set up a BRUSHED renderer. In the Rendering tab, create a Renderer Set containing one Renderer with mode set to BRUSHED. Import or create a brush in the Brush Library. In the renderer's Brushes tab, enable the brush and configure Stamp Spacing, Scale, and Opacity.
  6. Enable pressure influence. Still in the Brushes tab, set Pressure → Size to a value between 0 and 1.
    • 0.0 — pressure has no effect; all stamps are the same size.
    • 0.5 — stamp size is 50% pressure-driven. Light pressure = 50% of base scale; full pressure = full base scale.
    • 1.0 — stamp size is fully controlled by pressure. An anchor with pressure=0.05 will produce stamps at 5% of base scale (nearly invisible); pressure=0.9 gives 90% scale. This creates the strongest calligraphic effect.
    Optionally set Pressure → Opacity similarly to make lighter strokes more transparent.
  7. Assign the renderer set to the sprite and launch Loom (Cmd+L). Brush stamps along the open curve will now vary in size and/or opacity according to the drawn pressure profile.
Mouse input has no pressure variation. Drawing with a mouse always records pressure=1.0 at every anchor. To get pressure variation, you must use a pressure-sensitive tablet. If you see no size variation despite Pressure → Size being non-zero, verify the curve XML contains pressure attributes (values other than 1.0) on its anchor points.

Subdivision Tab

Subdivision is the core of Loom's pattern generation. The Subdivision tab defines Params Sets — named collections of subdivision stages that shapes reference.

Polygon types only: Subdivision applies exclusively to shapes whose source is a Polygon Set (spline or line) or a Regular Polygon. Shapes using Open Curve Sets or Point Sets bypass subdivision entirely and are rendered directly from their base geometry.

Structure

Params Set Collection
├── Params Set "mySet"
│   ├── SubdivisionParams #1  (stage 1: e.g. QUAD)
│   ├── SubdivisionParams #2  (stage 2: e.g. TRI_BORD_C, with transforms)
│   └── SubdivisionParams #3  (stage 3: e.g. SPLIT_VERT)
└── Params Set "anotherSet"
    └── SubdivisionParams #1  (single stage)

Each SubdivisionParams = one recursion level. Stages execute sequentially from first to last. The total polygon count grows multiplicatively: if stage 1 produces N₁ polygons per input, stage 2 produces N₂ per stage-1 polygon, and so on.

Use the Enabled checkbox to disable individual params without deleting them. Disabled params are written inside an <EditorDisabled> wrapper in the XML — the Scala engine ignores them, but they remain available to re-enable.

Subdivision Types — Complete Reference

Each SubdivisionParams specifies one type. Loom has two parallel sets of subdivision algorithms: LINE variants (for LINE_POLYGON type) and SPLINE variants (for SPLINE_POLYGON type). Both sets share the same type names in the UI.

Basic Division Types

TypePolygons per InputDescription
QUAD N (number of sides) Classic quadrilateral subdivision. Divides each polygon into N sub-polygons — one per original vertex. Each sub-polygon is formed from: the vertex, the midpoint of one adjacent side (controlled by Line Ratios), the polygon centre, and the midpoint of the other adjacent side. A square (4 sides) → 4 sub-polygons per stage. This is the most versatile and most commonly used type. Example: 3 QUAD stages on a square = 64 sub-polygons in a dense mosaic.
TRI N Triangular subdivision. Similar structure to QUAD but creates triangular sub-polygons using the vertex, midpoint of one adjacent side, and the centre. Produces more angular, triangular-feeling patterns. Particularly effective with triangle base polygons (producing fractal triangle hierarchies).
ECHO 2 Creates a scaled inner copy of the polygon directly inside the original. Produces a concentric nesting effect. The outer polygon and an inset copy sit inside each other. Control the gap with the Inset Transform scale.
ECHO_ABS_CENTER 2 Like ECHO, but the inner polygon's centre is calculated from the absolute centroid of all anchors rather than the local polygon centre. Useful for non-symmetrical shapes where the geometric centre differs from the visual centre.

Border Variants (QUAD)

TypeDescription
QUAD_BORD QUAD subdivision with a surrounding border. Produces the usual inner QUAD sub-polygons plus additional border polygons along each outer edge. Creates a two-zone pattern: a central subdivided area and an outer frame. The border adds N more polygons (one per side). Effective for creating panelling or mosaic tile effects with distinct borders.
QUAD_BORD_ECHO QUAD_BORD where the border polygons themselves are echoed (concentric inner copy added). Combines the border effect with a nesting echo inside each border polygon.
QUAD_BORD_DOUBLE Like QUAD_BORD but creates a double border — two rings of border polygons around the central subdivision. Produces richer framing effects.
QUAD_BORD_DOUBLE_ECHO QUAD_BORD_DOUBLE where the border polygons also have echo copies inside them.

Border Variants (TRI)

TypeDescription
TRI_BORD_B Triangular subdivision with a border. Similar to QUAD_BORD but using the TRI algorithm for the central subdivision. The border polygons run along each outer edge.
TRI_BORD_B_ECHO TRI_BORD_B with echo copies inside the border polygons.
TRI_BORD_C A second variant of TRI border subdivision with a different geometric construction for the border polygons. Produces a distinct border shape compared to TRI_BORD_B.
TRI_BORD_C_ECHO TRI_BORD_C with echo copies inside the border polygons.
TRI_STAR Star-shaped triangular subdivision. Produces sub-polygons radiating from the centre in a star pattern. Particularly striking with triangular base polygons, producing a Star-of-David-like recursive structure.
TRI_STAR_FILL TRI_STAR with a central fill polygon added to cover the central gap. Prevents the hollow centre typical of the star arrangement.
TRI_BORD_A Corner-triangle subdivision (N output polygons). Creates one triangle per corner of the input polygon, each spanning that corner and the midpoints of its two adjacent outer sides. Unlike TRI_BORD_B, no inset polygon is used — midpoints are sampled directly from the outer sides via Line Ratios. The result is small triangles at each corner with unfilled space between them. Example: applied to a square, produces 4 corner triangles whose bases cut diagonally across the sides.
TRI_BORD_A_ECHO TRI_BORD_A with an additional star-shaped echo polygon (N+1 output polygons). The echo alternates between the original corner vertices and the outer side midpoints — corner[0] → outerMid[0] → corner[1] → outerMid[1] → … — producing a zigzag star outline overlaying the N corner triangles. The echo is the last polygon in the output array and can be isolated or hidden using Visibility Rules.

Split Types

TypePolygons per InputDescription
SPLIT_VERT 2 Splits each polygon into two halves with a vertical cut (left/right). The algorithm detects the most horizontal pair of opposite sides and divides there, so the split is always genuinely vertical even after previous subdivision stages have rotated or distorted the shapes. Example: 4 stages of SPLIT_VERT on a square → 16 tall thin strips.
SPLIT_HORIZ 2 Splits each polygon into two halves with a horizontal cut (top/bottom). Geometry-aware: finds the most vertical pair of sides. Example: alternating SPLIT_VERT and SPLIT_HORIZ stages creates a brick-like grid.
SPLIT_DIAG 2 Splits each polygon diagonally. Finds the pair of sides whose connecting line is most diagonal (closest to 45°). Creates triangular-feeling pairs. Mixing with QUAD produces angular, faceted results.
SPLIT types always produce exactly 2 polygons per input, regardless of the polygon's number of sides. They are geometry-aware: even after many recursion levels distort the shapes, SPLIT_VERT will still detect and cut the most horizontal sides.

Basic Parameters

ParameterUnitDescription
Random MiddleBooleanRandomise the centre point of each polygon before subdivision. Creates organic distortion. Off: clean geometric subdivision. On: each polygon's centre drifts randomly, making the subdivision messy and organic.
Random DivisorIntegerControls the degree of centre randomisation. The random offset range is proportional to the polygon size divided by this value. Low values (2–10): dramatic, distorting shifts. High values (50–200): subtle wobble. Requires Random Middle to be on.

Point-Level Transforms

These manipulate specific sets of points within each sub-polygon. They are the most powerful tools for organic distortion in Loom. Five transform types are available; each targets a different subset of the polygon's points.

Exterior Anchors

Moves the non-centre anchor points — the vertices that sit on the edges of the parent polygon. Primary effect: spiking.

ParameterUnitDescription
Spike FactorFractionAmount to move exterior points toward or away from the polygon centre. 0.0: no movement. −0.5: push 50% of the distance from centre outward → spiky, star-like protrusions. +0.5: pull 50% toward centre → pinched, convex shapes. −1.0 or lower: extreme spikes well beyond the parent boundary.
Which SpikeEnumALL: all exterior anchors. CORNERS: only anchors that correspond to original polygon corners (creates sharper corner emphasis). MIDDLES: only midpoint anchors on original sides (creates mid-edge bulging).
Spike TypeEnumSYMMETRICAL: sibling pairs (adjacent anchors) move equally in mirrored directions — creates symmetric pinching. RIGHT: only the right sibling moves. LEFT: only the left sibling moves. RANDOM: random direction each frame.
Spike AxisEnumXY: full 2D movement. X_ONLY: horizontal movement only (creates vertical streaking). Y_ONLY: vertical movement only (horizontal streaking).
CPs FollowBooleanWhen enabled, the bezier control points associated with each moved anchor shift by the same amount. Keeps curves anchored to the moved point, preserving smooth connections. When disabled, control points stay put — creates kinked, angular junctions.
CPs SqueezeBooleanVary the perpendicular distance between control points, creating sharper or more bulbous curves at the anchor positions.

Central Anchors

Moves the centre point(s) of sub-polygons and their associated control points. Primary effect: tearing — pulling the centre apart to create openings or folded-over overlaps.

ParameterUnitDescription
Tear FactorFraction of side lengthHow far to pull the centre point from its geometric position. 0.1: subtle wobble in the centre. 0.5: pronounced opening. 1.0+: centre pulled far outside the polygon — extreme overlapping forms.
Tear AxisEnumXY, X_ONLY, Y_ONLY, RANDOM. Constraining to one axis creates directional streaks.
Tear DirectionEnumDIAGONAL: moves outward from the polygon's centroid. LEFT: toward the next exterior anchor. RIGHT: toward the previous exterior anchor. RANDOM: changes each frame.
All Points FollowBooleanAll other points in the polygon shift in the same direction as the centre movement. Creates a translation of the entire polygon rather than just a local distortion of the centre region.
Inverted FollowBooleanWhen All Points Follow is on, the remaining points move in the opposite direction — creating an expanding/tearing-apart effect where centre and extremities move away from each other.

Inner Control Points

Fine-tunes bezier control points along the internal edges running from exterior anchors to the centre. Primarily affects the curvature of the internal edges and how they join at the centre.

ParameterDescription
Refer To OuterFOLLOW_OUTER: inner control points mirror the curvature of the outer edge, creating smooth transitions. EXAGGERATE_OUTER: amplifies the outer curvature inward. COUNTER_OUTER: reverses the outer curvature — creates S-curves. NONE: control points placed at default positions.
Inner Multiplier / Outer MultiplierScale factors for control point positions along the inner lines. Higher values push control points further along toward the centre or exterior anchor.
Common LineWhether adjacent internal lines share control point positions. Enabling this prevents overlapping shapes along shared internal edges.

Outer Control Points

Controls the curvature of the external (non-centre-connecting) edges of sub-polygons. Two approaches:

  • Perpendicular curving — Pushes control points perpendicular to the edge line, creating puff or pinch effects. Curve types include: PUFF (both control points curve outward), PINCH (both curve inward), PUFF_PINCH (first outward, then inward — S-curve), PUFF_PINCH_PUFF_PINCH (alternating pattern for longer edges).
  • Curve from centre — Uses the polygon's absolute geometric centroid as a reference. Effective for creating wave-like distortions, especially with asymmetric or irregular base shapes.

Anchors Linked to Centre

Acts on the anchor points at the ends of lines connecting outer edges to the polygon centre — i.e. the points where the internal spokes meet the outer boundary. Primary effect: tearing these anchors toward corners, the opposite corner, or the centre itself. Useful for creating "fan" and "burst" patterns.

Point-level transforms compound across recursive stages. A Spike Factor of −0.1 (10% outward) at every stage over 5 recursion levels produces dramatically spiked results at the finest level. Start with very small values and add stages incrementally to see how the effect builds.

Animation

Sprite animation controls how a sprite transforms over time when Animating is enabled (Run tab). Two main animation paradigms are supported: Random (Jitter) and Keyframe.

Random (Jitter) Animation

Each animation frame, random perturbations are applied to the sprite's scale, rotation, and position within specified ranges. This creates organic, continuously varying motion with no fixed path.

ParameterUnitDescription
Scale FactorMultiplier/frameBase scale change applied every frame. 1.0: no change. 1.002: grows 0.2% per frame (after ~350 frames, double-size). 0.999: slowly shrinks.
Rotation FactorDegrees/frameBase rotation change per frame. 0: stationary. 1.0: rotates 1° per frame, making one full revolution in 360 frames. −0.5: slow counter-clockwise rotation.
Speed Factor X / YLocation units/frameBase translation per frame. Small values (0.1–0.5) create slow drifting; larger values (2.0+) produce fast movement across the canvas.
Scale RangeMultiplierMaximum random variation around Scale Factor. If Scale Factor = 1.002 and Scale Range = 0.001, the actual per-frame scale varies randomly from 1.001 to 1.003. Creates pulsing/breathing effects.
Rotation RangeDegreesMaximum random variation around Rotation Factor. Creates wobbly, unsteady rotation.
Translation RangeLocation unitsMaximum random variation around Speed Factor. Creates jittery, trembling motion.
For animation that looks "alive" without drifting off-screen, use small base factors (1.0, 0, 0) with moderate ranges. The sprite will oscillate around its starting position without permanently moving away.

Keyframe Animation

Keyframes define explicit states (position, scale, rotation) at specific draw cycle numbers. Loom interpolates between consecutive keyframes using the easing algorithm specified for each transition.

Keyframe Parameters

ParameterUnitDescription
Draw CycleIntegerThe animation frame number when this keyframe occurs. Must be in ascending order. Keyframe 0 is the initial state; keyframe 60 is reached after 60 animation frames.
Position X / YLocation percentage deltaPosition change to apply from the previous keyframe to this one. Cumulative: each keyframe's position is added to the previous. Example: keyframe 0 at (0,0), keyframe 60 at (20,0) moves the sprite 20% to the right over 60 frames.

RotationDegrees deltaRotation change from previous keyframe. 90 = rotate 90° clockwise to reach this keyframe. Accumulates: a sequence of 90° keyframes produces continuous spinning.
EasingEnumThe interpolation algorithm used to transition from this keyframe to the next. See Easing Types below.

ModeDescription
LOOPWhen the last keyframe is reached, the animation immediately restarts from the first keyframe. Good for continuous, seamlessly repeating motion.
PING_PONGWhen the last keyframe is reached, the animation reverses and plays backward to the first keyframe, then reverses again. Creates back-and-forth oscillation without a visible jump cut.

Morph Animation

Morph animation blends a sprite's vertices between the base shape and one or more morph targets frame by frame. Targets form a chain: base → mt1 → mt2 → … Each morph target must have the exact same topology as the base shape (same number of polygons/curves, same number of vertices, same polygon type).

Both polygon sets and open curve sets support morph animation. The morph target file type must match the base shape type — use .poly.xml target files for polygon shapes, .curve.xml target files for open curve shapes.

Point sets do not support morph animation. The morph controls are disabled for sprites whose shape source is a Point Set.

Morph amount semantics — 0 to N

The morphAmount value is a continuous chain position from 0 to N where N is the number of morph targets:

ValueMeaning
0.0Base shape (no morphing)
1.0Fully morphed to first target (mt1)
2.0Fully morphed to second target (mt2)
1.5Halfway between mt1 and mt2
NFully at the Nth target

Single-target sprites with morphAmount in the range 0–1 behave identically to the previous single-target implementation.

Morph target file naming

Files are stored in the project's morphTargets/ directory and identified by suffix:

SuffixGeometry typeExample
.poly.xmlPolygon set (<polygonSet> root)myshape_mt_1.poly.xml
.curve.xmlOpen curve set (<openCurveSet> root)myshape_mt_1.curve.xml

Both types can coexist in the same morphTargets/ directory. Files without a recognised suffix are treated as polygon format for backward compatibility.

Adding morph targets in the Sprites tab

  1. Select a sprite in the Sprites tab whose shape source is a Polygon Set or Open Curve Set.
  2. In the Animation panel, choose Jitter Morph or Keyframe Morph.
  3. In the Morph Targets panel, click Add to open a file browser pointing at morphTargets/. Select a .poly.xml or .curve.xml file.
  4. The target appears in the list. Its position in the list determines its chain index (first entry = mt1, second = mt2, etc.).
  5. Repeat to add further targets. Use ↑ / ↓ to reorder or Remove to delete.
  6. Click Create Copy to generate a new morph target file as an editable copy of the base polygon — this creates the initial file in morphTargets/ for you to edit in Bezier Draw.
If the topology warning dialog appears when adding a target, the vertex or polygon count does not match the base shape. Adding it anyway will cause Loom to fall back to the base shape at runtime. Fix the target's vertex count in Bezier Draw before using it.

Step-by-step: morph chain animation workflow

  1. Create the base shape — draw or import a polygon set in Bezier Draw; register it in the Geometry tab.
  2. Create the shape and sprite — configure a Shape and Sprite using the base geometry as normal.
  3. Set the animator to Keyframe Morph — select the sprite, choose Keyframe Morph in the Animation panel.
  4. Add keyframes — add at least two keyframes. Set Morph Amount on each: e.g. keyframe 0 → 0.0, keyframe 60 → 1.0, keyframe 120 → 2.0 to sweep through two targets.
  5. Add morph target files — click Add in the Morph Targets panel; select the .poly.xml or .curve.xml files in order.
  6. Reload Loom (Cmd+R) — the animation sweeps base→mt1→mt2 over the specified draw cycles.

Brush Renderer (BRUSHED Mode)

The BRUSHED renderer mode stamps greyscale brush PNG images along polygon or open curve edges, or at individual point positions. This enables ink-wash, charcoal, watercolour, hand-drawn stroke effects, and — by using point sets — placement of bitmap images at specific canvas locations.

How It Works

  1. Loom extracts all edges (for polygon/curve types) or point positions (for point sets).
  2. Brush stamps from greyscale PNGs in the project's brushes/ directory are placed at intervals along edges or at each point position.
  3. Stamps are tinted by the renderer's Stroke Color before compositing onto the canvas.
  4. In PROGRESSIVE mode, edges are revealed gradually; in FULL_PATH mode all edges are redrawn every frame. Point sets always use single-stamp placement.

Meander Path New

The Meander Path feature causes brush stamps to follow a gently wandering path between each pair of anchor points, rather than exactly tracing the underlying polygon or curve geometry. The effect ranges from subtly hand-drawn (small amplitude) to urgently scribbled (large amplitude) to organically breathing over time (animated mode).

Accessing Meander Settings

Meander is available on any renderer in BRUSHED mode. In the renderer editor, open the Brushes tab. At the bottom of the brush settings is a collapsible Meander Path section:

  • Click ▶ Meander Path to expand or collapse the settings panel. The panel collapses fully when not needed, giving the brush preview its maximum space.
  • The Enabled checkbox sits beside the toggle button and activates the feature independently of whether the panel is open or closed. You can collapse the settings and leave Enabled checked — the feature stays active.

How It Works

The algorithm operates per edge (each segment from one anchor point to the next):

  1. A smooth noise function is sampled at evenly spaced positions along the edge to produce perpendicular displacement values.
  2. A sin(πt) taper envelope multiplies the displacement at each position — it is zero at both endpoints (t=0 and t=1) and peaks at the midpoint (t=0.5). This forces the wandering path to return cleanly to the original anchor points at both ends, ensuring seamless joins between consecutive edges.
  3. The displaced sample points are joined with a smooth Catmull-Rom spline to form the perturbed path.
  4. Brush stamps are placed at the configured stamp spacing along this perturbed path. If Follow Tangent is enabled, stamp rotation tracks the meander tangent rather than the original edge direction.
  5. Each edge gets a distinct meander shape derived from its index in the polygon — adjacent edges wander differently without configuration, producing natural variety across the whole shape.

Meander Parameters

ParameterUnitDescription
AmplitudePixels Maximum perpendicular displacement from the original edge line.
  • 2–5 px: subtly hand-drawn, barely distinguishable from straight unless looked at closely
  • 10–20 px: clearly organic, sketchy, ink-on-paper quality
  • 30–60 px: dramatically scribbled, expressive mark-making
  • 80+ px: wildly loose; strokes detach significantly from the underlying geometry
When Scale Image is enabled (Run tab), amplitude is multiplied by the Quality Multiple so marks remain proportionally consistent at high resolution.
Frequency0.001–0.5 Controls how tightly the meander oscillates along the edge — roughly, the number of noise cycles per unit length.
  • 0.01–0.03: wide, lazy curves — the path swings gently across the edge
  • 0.05–0.10: moderate back-and-forth within each edge
  • 0.15–0.30: rapid oscillation, producing a trembling or hatching-like quality
High frequency combined with high amplitude creates a coiled, frantic look; low frequency with high amplitude creates broad sweeping detours.
SamplesInteger (4–200) Number of sample points placed along each edge to define the perturbed path. More samples = smoother result, particularly on long or highly curved edges.
  • 12–24: good for most uses
  • 32–48: smoother on very long edges or at high quality multiples
  • 4–8: minimal but fast — angular, faceted meander quality
Seed (0=auto)Integer Controls the random character of the noise pattern.
  • 0 (auto): a unique seed is derived automatically from each edge's index in the polygon. Adjacent edges get different seeds — natural variety with no configuration needed. The pattern is consistent across reloads with the same geometry.
  • Non-zero: sets a specific base seed. Use this to lock in a particular meander character and reproduce it exactly. Different non-zero values give distinctly different noise patterns.

Animated Meander (Full Path Draw Mode)

When enabled, the noise pattern shifts gradually each frame, causing the meandering path to drift and writhe continuously. This creates a living, breathing quality to the strokes — as if drawn by a slowly trembling hand or as ink diffusing in water.

Full Path mode only. Animated meander has no effect in Progressive draw mode. In Progressive mode, the meander path is computed once at the start of the reveal with a fixed seed, and the path does not change during the progressive draw. To use animated meander, set Draw Mode to Full Path, which redraws the entire path every frame.
ParameterDescription
Animated (Full Path only) Activates the per-frame phase shift. When checked (and Draw Mode = Full Path), the meander noise advances each frame, making the path drift over time.
Speed How far the noise pattern shifts per frame.
  • 0.001–0.003: slow, meditative drift — the path barely moves between frames
  • 0.005–0.015: gentle, noticeable flow — a good starting point
  • 0.03–0.06: lively, energetic movement
  • 0.1+: fast, restless; can feel noisy rather than fluid at very high values
For a stroke that feels alive but not distracting, try Amplitude=10, Frequency=0.03, Animated=on, Speed=0.005 with Full Path draw mode. The path will drift so slowly that it is almost subliminal — visible over several seconds of animation but not jarring frame to frame.

Scale Along Path

An independent noise envelope can vary stamp scale continuously along the meandering path, creating additional organic mark-size variation. This is separate from and complementary to the random Scale min/max range in the brush settings — Scale Along Path replaces the per-stamp random scale when enabled, replacing it with a spatially coherent pattern.

ParameterDescription
Scale Along Path Enable scale variation along the path. When checked, each stamp's scale is driven by the scale envelope rather than being drawn randomly from the Scale min/max range.
Freq How many scale oscillations occur per unit length along the edge. Controls how rapidly the mark size swells and contracts.
  • 0.01–0.03: broad, smooth swells — marks grow and shrink slowly along the stroke
  • 0.05–0.10: moderate variation
  • 0.2+: rapid alternation, creating a beaded or dotted-line quality
Range 0.0–1.0. How far the scale deviates from the midpoint between the renderer's Scale Min and Scale Max.
  • 0.0: no variation — all stamps at the midpoint scale
  • 0.5: moderate swells, reaching 50% of the way toward min/max
  • 1.0: full range — stamps vary from Scale Min to Scale Max over the path length
Scale Along Path combined with meander amplitude gives a very convincing hand-drawn brush stroke: the path wanders and the mark pressure varies together. Set Scale min=0.6, max=1.4, Scale Along Path enabled, Range=0.8, Freq=0.04 alongside Amplitude=12 for an ink-brush quality.

Step-by-step: setting up an organic ink-stroke effect

  1. Create or import a polygon set or open curve set in the Geometry tab. Open curves work particularly well for ink-stroke aesthetics.
  2. In the Rendering tab, create a renderer set with a single renderer. Set its mode to BRUSHED.
  3. In the Brushes tab, import or create a brush. A thin oval or calligraphy-shaped brush gives the best ink results. Set Stamp Spacing to 4–8, Follow Tangent on, Scale 0.8–1.2, Opacity 0.5–0.9.
  4. Set Draw Mode to Full Path.
  5. Click ▶ Meander Path to expand the settings. Check Enabled.
  6. Set Amplitude=10, Frequency=0.03, Samples=24, Seed=0.
  7. Optionally check Animated and set Speed=0.005 for a slowly shifting stroke.
  8. Create a shape and sprite using this renderer. Press Cmd+L to launch Loom. The strokes follow a gently wandering organic path rather than the exact Bézier curves.
  9. Press Cmd+R after any parameter change to reload without restarting Loom.

Pressure Sensitivity New

Loom supports end-to-end stylus pressure, from the moment you draw an open curve in Bezier Draw through to brush stamp size and opacity variation in the rendering engine. This enables calligraphic, ink-brush, and expressive mark effects where heavier stylus pressure produces larger, more opaque stamps and lighter pressure produces finer, more delicate ones.

How the Pipeline Works

Tablet stylus → Bezier Draw
pressure recorded per anchor → openCurveSet XML
pressure="0.xxx" attribute → Scala engine load
Polygon2D.pressures[] → BrushEdge
pressureStart / pressureEnd → Stamp engine
scale & opacity per stamp

The Pressure Formula

At each stamp position along an edge, the engine linearly interpolates pressure between the start and end anchors of that segment, then applies:

pressure = lerp(edge.pressureStart, edge.pressureEnd, t)

stampScale   = baseScale   × (1 − pressureSizeInfluence   + pressure × pressureSizeInfluence)
stampOpacity = baseOpacity × (1 − pressureAlphaInfluence  + pressure × pressureAlphaInfluence)

At influence=0, the pressure term drops out and stamps use only the random base scale/opacity. At influence=1, stamps scale linearly with pressure from near-zero (at pressure=0) up to the full base scale (at pressure=1.0).

Step-by-Step: Creating a Pressure-Sensitive Brush Stroke

  1. Draw with a stylus. Launch Bezier Draw from the Curves sub-tab → Create in Bezier. Draw open curves using a pressure-sensitive stylus (Wacom or similar tablet). Vary pressure as you place each anchor — heavy for thick parts, light for fine tapering ends.
  2. Verify the pressure profile. Switch to Point Selection Mode (a) in Bezier Draw. Anchor points are drawn at a size proportional to their recorded pressure — large filled circles for high pressure, tiny dots for low. If all dots are the same size you are using a mouse; a tablet is required.
  3. Save. Click Save in Bezier Draw. The file is written to curveSets/{name}.xml with pressure="0.xxx" on each anchor point element. Close Bezier Draw.
  4. Register the curve set in the Curves sub-tab. The dropdown auto-selects the new file after Bezier closes.
  5. Create Shape and Sprite referencing the curve set (source type: Open Curve Set).
  6. Set up a BRUSHED renderer. In the Rendering tab, set the renderer mode to BRUSHED. Import or create a brush that suits the stroke style (a narrow oval, tapered brush, or brush with varying opacity works best).
  7. Configure pressure influence. In the renderer's Brushes tab:
    • Set Pressure → Size to 1.0 for maximum calligraphic effect. Values of 0.5–0.8 give a more subtle modulation.
    • Optionally set Pressure → Opacity to 0.3–0.7 so lighter strokes are also more transparent.
    • Keep Scale min/max close together (e.g. 0.9–1.1) when using full pressure influence, since the pressure curve already controls size variation.
  8. Launch Loom (Cmd+L). Brush stamps along the stroke will vary dramatically — nearly invisible where you drew lightly, full-size and bold where you pressed hard.
  9. Iterate. Press Cmd+R to reload parameters without restarting. Adjust Pressure → Size and base Scale values to fine-tune the feel.
