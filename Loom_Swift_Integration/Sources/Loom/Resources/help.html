What is Loom?

Loom is a generative art environment for creating still images and animations driven by geometric subdivision, procedural rendering, and keyframe animation. It is designed for visual experimentation — each project is a composition of shapes, subdivision rules, and rendering styles that combine to produce imagery ranging from technical line drawings to richly textured organic forms.

Distinguishing features

Subdivision engine
20+ algorithms — Quad, Tri, Split, Echo, Star — each with ratio, inset, and per-wall colour controls.
Animated rendering
Custom brush and stamp PNG patterns, stencil masking, progressive drawing, and colour, width, and opacity animation on every renderer.
Procedural drivers
Noise, oscillator, jitter, and keyframe drivers animate sprite transforms, sprite opacity, camera motion, and renderer-level fill, stroke, width, and opacity.
Morph targets
Smoothly interpolate between two geometry states with an animated morph-amount driver.
Sprite hierarchy
Parent–child sprite trees with selectable inherit masks for position, rotation, and scale.
Pressure sensitivity
Freehand and post-traced pressure data can influence brush opacity and stroke weight, and can be carried into subdivided children.
Non-destructive bake
Preview subdivision live, then bake to a permanent polygon file when satisfied.

Core workflow

  1. Create or open a project. The project folder holds all geometry files, brushes, stamps, and the project_config.json.
  2. In the Geometry tab, draw or import polygon, open-curve, and point geometry using the built-in editor.
  3. In the Sprites tab, create sprite instances and assign your geometry. Set position, scale, and rotation. Establish parent–child hierarchies if needed.
  4. In the Subdivision tab, define sets of subdivision parameters and assign them to sprites. Preview the subdivided result and bake when ready.
  5. In the Rendering tab, create renderer sets — choose fill/stroke modes or brush/stamp patterns. Assign each renderer set to the sprites that should use it.
  6. Add animation via the Sprites inspector, Rendering inspector, or the Timeline. Use drivers for sprite transforms, camera motion, renderer colours, stroke width, and renderer opacity.
  7. Press Play (Space) to preview. Export as a still, SVG, or video from the toolbar.

Quick Start

New project

  1. Choose File → New Project… (or ⌘N). Enter a name in the dialog and click Create.
  2. Loom creates the project folder and opens project_config.json with sensible defaults (800×800 canvas, 30 fps).
  3. The window switches to the main layout — the five tab panels and playback toolbar become active.

Open an existing project

  1. Choose File → Open Project… (or ⌘O) and select the project folder, or click the Open Loom Projects Folder icon in the toolbar and double-click a project.
  2. Loom reads project_config.json and reloads all geometry, renderers, and animation state.
  3. Use ⌘R or the reload button to re-read the config after making external edits.

Geometry Tab

The Geometry tab manages the polygon and curve files that define all shapes in your project. A geometry document holds one or more named layers, each containing polygons, curves, and points.

Geometry list

The left panel shows all registered geometry sources. Each row displays the file name and type indicators (polygon, curve, point icons) alongside a sprite-count badge. Click to select; double-click to rename. Right-click for Rename, Duplicate, and Delete options.

Tip: Geometry files are shared — multiple sprites can reference the same file with different subdivision sets and renderers applied.

Quick setup

Quick setup creates a working pipeline from a geometry source without requiring separate dialogs for every hidden shape and shape-set object. Shape sets and shapes still exist internally, but the user-facing workflow concentrates on geometry, subdivision, sprites, render sets, and renderers.

  • Suggested names are generated automatically using concise suffixes: SubSet, SprtSet, RenSet, and Ren.
  • Subdivision, sprite, renderer set, and renderer fields are dropdowns. You can accept the suggested name or choose an existing item in the project.
  • If the geometry is not subdividable, the subdivision field shows None. Polygonal geometry can also be deliberately assigned None when subdivision is not wanted.
  • Make Pipeline creates the necessary behind-the-scenes links but leaves you in the current tab. A visual status signal confirms that the pipeline has been built.

Selection behavior

  • The edit modes are mutually exclusive: points, edges, open curves, and polygons. The active mode is highlighted.
  • Hold Shift while clicking to add to the current selection. Hold ⌘ while clicking to toggle/remove from it.
  • Point clicks are stack-aware: when overlapping welded points sit in the same location, selecting one selects the stack so mesh corners stay coherent.
  • Rubber-band point selection captures all points in the rectangle, including overlapping welded stacks. Rubber-band edge selection selects edges by their midpoints.
  • For point and edge groups, movement begins only when the drag starts inside the overall selected bounds. Clicking outside those bounds drops the selection unless you drag a new rubber-band rectangle.
  • Scaling and rotation use the same selected geometry path as canvas dragging, so whole objects, edges, and point groups can all be transformed.

Layers

Each geometry document supports multiple named layers. Use layers to separate shape groups — for example, keeping outline polygons on one layer and detail curves on another.

  • Click a layer to make it active. By default this gives that layer focus: it remains editable and fully coloured, while other layers become visible but non-editable grey context.
  • Toggle the pencil icon to make an additional non-selected layer editable. This is useful when intentionally working across multiple layers.
  • Toggle the eye icon to show or hide a layer entirely.
  • Double-click a layer name to rename it.
  • Add, rename, duplicate, delete, and reorder layers with the layer controls.
  • Layered JSON is the source of truth. Pipeline targets can still address individual layers for subdivision, sprite, and rendering setup without losing the full editable layered document.

Pressure sensitivity

Loom can store pressure samples on freehand and traced geometry. Pressure data is used by SVG export and brush rendering to create hand-drawn variation in stroke weight and opacity. Geometry that contains pressure data is visually marked in the editor so it is clear which shapes carry pressure information.

Creating pressure data

  • Freehand drawing records pressure automatically when the input device provides it. If no device pressure is available, Loom still records a usable default profile.
  • Pressure Trace lets you post-add pressure to geometry created by points or point-by-point tools. Select a shape or edge, activate Pressure Trace, then trace along the relevant path.
  • Pressure tracing samples densely along the path so point-by-point geometry can acquire convincing variation rather than a simple straight interpolation.
  • Pressure can be discarded from selected geometry and re-traced if the result is not right.

Closed pressure-sensitive shapes

Freehand strokes can be finalised as closed polygons while retaining their pressure profile. Pressure remains attached to the path rather than being discarded when the shape becomes a closed polygon. This makes it possible to draw closed, pressure-sensitive regions and then feed them into subdivision and rendering.

Pressure and subdivision

When a pressure-sensitive polygon is subdivided, children can receive pressure in different ways from the Subdivision tab. See Subdivision Pressure for the current modes.

Sprites Tab

Sprites are the fundamental render units in Loom. Each sprite references a geometry file and is assigned a subdivision set and a renderer set. The Sprites tab shows a hierarchical tree of all sprites organized into named sets.

Sprite list

Sprites are grouped into sprite sets. Click the chevron to collapse or expand a set. Drag sprites within the list to reorder them, or drag between sets to reassign. Use the search bar at the top to filter by sprite or set name.

Each sprite row shows:

  • An enable checkbox — disabled sprites are not drawn.
  • The sprite name (double-click to rename).
  • A play icon if any animation driver is active.
  • For child sprites, P / R / S buttons to toggle parent transform inheritance.

Creating sprites

  1. Click + in the set toolbar to add a new sprite set, or use the sprite + button to add a sprite to the selected set.
  2. With the new sprite selected, open the inspector and assign a Geometry (polygon set file), a Renderer Set, and a Subdivision Set.
  3. Set initial Position, Scale, and Rotation in the Transform section.

Hierarchy and inheritance

Any sprite can be assigned a parent via the Hierarchy section in the inspector. Child transforms are applied relative to the parent. Use the P / R / S toggles to choose which parent transforms the child inherits:

ToggleEffect when on
P (Position)Child offset is applied relative to the parent's world position.
R (Rotation)Child rotation is added to the parent's world rotation.
S (Scale)Child scale is multiplied by the parent's world scale.
Tip: Set all three off to create a sprite that uses the parent only for logical grouping, with completely independent world-space transforms.

Depth & Parallax

Each sprite has a Depth value in its Transform section. Combined with the Perspective setting in the Global tab Camera section, depth creates a 2.5D parallax effect: sprites at different depths move at different rates when the camera pans or zooms, giving a sense of three-dimensional space without changing the underlying 2D geometry.

How depth works

The depth value scales each sprite's effective camera zoom and pan offset by a depth factor: f = 1 / (1 + depth × perspectiveStrength). A sprite at depth 0 always uses the camera transform unchanged (the focal plane). Positive depth values make sprites appear smaller and lag behind camera movement; negative values make them appear larger and move more than the camera.

Draw order

Sprites are automatically drawn back-to-front based on their depth — the sprite with the greatest depth value is drawn first (furthest away), and the most negative depth is drawn last (on top). This means you do not need to manually order sprites to get correct occlusion between depth layers.

Quick setup

  1. In the Global tab, set Camera → Enabled on and enter a Perspective value — start with 0.003.
  2. In the Sprites tab, select each sprite and set its Depth in the Transform section. Use 0 for the focal-plane layer, positive values (e.g. 3, 6) for background sprites, and negative values (e.g. -1) for foreground sprites.
  3. Animate the camera Pan X/Y in the Timeline. Background sprites will drift more slowly than focal-plane sprites, creating a parallax sense of depth.
Tip: Camera zoom animation also creates depth — zooming in makes background sprites grow more slowly than focal-plane sprites, giving a push-in feel. Combine with pan for a full parallax camera move.

Subdivision Tab

The Subdivision tab is where Loom's generative power lives. It manages subdivision parameter sets — ordered sequences of algorithms that progressively fracture geometry into more complex polygonal structures.

Working with sets

  1. Click + in the Sets toolbar to create a new subdivision set and give it a name.
  2. Click + in the Parameters toolbar to add the first algorithm to the set.
  3. Select the parameter row and adjust its settings in the inspector on the right: algorithm type, ratios, inset, visibility rules, pressure handling, and optional PTW/PTP geometric transforms.
  4. Click the eye icon next to the set to preview the subdivided result on the canvas — the sprites assigned to this set update live.
  5. When satisfied, click Apply in the Apply Bar to commit the set assignment to the selected sprite, or use the bake button to write the subdivided geometry to a permanent file.

Set and parameter controls

  • The bottom toolbar is grouped into set controls, parameter controls, and bake/export controls.
  • Subdivision parameters can be renamed with the angled pencil icon. Concise names make long recursive chains easier to read.
  • PTW and PTP sections stay collapsed until opened. PTP remains disabled until explicitly enabled, preventing accidental point-level deformation.
  • The SVG button exports a wireframe vector file without baking; the bake button writes static subdivided geometry back into polygonSets/.

PTW — Polygons Transform Whole

PTW applies geometric transformations to entire output polygons as units. When first enabled, geometry remains at its existing 100% scale until you deliberately adjust the transform values. Each polygon produced by the subdivision step can then be independently translated, scaled, and rotated by a randomly sampled amount within configurable ranges. This breaks the regularity of subdivided geometry, producing scattered, fragmented, or organically varied results from otherwise structured forms.

ParameterDescription
EnabledMaster on/off toggle for the entire PTW block.
WholeWhen on, the transform is applied to the whole polygon shape as a rigid body. When off, only positional displacement is applied.
ProbabilityPercentage chance (0–100) that the transform fires for each individual polygon. At 50%, roughly half the output polygons will be affected each frame.
Common centreWhen on, all transforms use a single shared centre point rather than each polygon's own centroid. Useful for creating radial scatter effects.
Ran translEnable random translation. When off, no positional displacement is applied even if translation ranges are set.
Ran scaleEnable random scaling.
Ran rotateEnable random rotation.
Transl X (mn/mx)Min/max range for random displacement along the X axis.
Transl Y (mn/mx)Min/max range for random displacement along the Y axis.
Scale X (mn/mx)Min/max range for random X-axis scale factor.
Scale Y (mn/mx)Min/max range for random Y-axis scale factor.
Rot range (mn/mx)Min/max range for random rotation angle.
Tip: Set Probability below 100 and Whole off with a small Transl range to add subtle positional noise to subdivided shapes without destroying their overall structure.

PTP — Polygons Transform Points

PTP operates at the individual point level inside each output polygon. Rather than moving whole shapes, it deforms specific sets of Bézier anchor points and control handles — creating spikes, tears, curvature changes, and organic distortions. PTP is organised into five independent groups accessed via tabs, each targeting a different class of points within the polygon.

All five groups share a top-level Enabled toggle and Probability (applied per polygon). Each group has its own secondary probability.

Ext — Exterior Anchors

Displaces the outer anchor points of each polygon outward or inward — creating spikes, dents, or irregular protrusions. The target set can be all anchors, corners only, or midpoint anchors. Spike direction can be symmetrical, left-biased, right-biased, or random. Control point handles can optionally follow the displaced anchor with a configurable multiplier, and a squeeze option pulls handles toward the edge midpoint for sharper transitions.

Spike axisConstrain displacement to XY (both), X, or Y only.
Ran spike / Spike rangeRandomise spike magnitude within min/max range.
CPs follow / CPs multControl handles track the displaced anchor. Multiplier scales how closely they follow.
Ran CPs follow / CPs follow rangeRandomise the CP follow amount per anchor.
CPs squeeze / Squeeze factorPull control handles toward the edge midpoint, sharpening the spike base.

Tears or displaces anchor points positioned near the polygon interior, pulling them in a specified direction and axis. An all points follow option propagates the tear displacement to the rest of the polygon's points, effectively shearing the whole shape. Inverted follow reverses this propagation direction.

Repositions the Bézier control handles on the exterior edges of the polygon to introduce curvature. Handles can be oriented perpendicular to the edge, or directed from the polygon centre outward. The curve type controls whether individual edges bow outward (puff) or inward (pinch), and how these alternate around the polygon perimeter.

Pressure subdivision

The Pressure section controls how pressure data from a pressure-sensitive source polygon is passed to children created by subdivision. If the source geometry has no pressure information, Loom skips the extra pressure mapping work.

ModeBehaviour
NoneDo not pass pressure data to children. Useful when pressure should affect only the source outline.
SpatialMap child pressure from the child polygon's position within the parent. Faint or heavy regions of the parent influence children in the same spatial area.
Inherit PathEach child inherits the parent's pressure path as a reusable profile, preserving the overall hand-drawn rhythm rather than remapping by location.
RandomAssign pressure profiles from a library of sampled line-pressure behaviours. Use the 1–5 checkboxes to choose which pressure groups are allowed, from light/whispy through firm/vigorous.
Pressure subdivision is most useful with brush rendering, where pressure can vary stamp opacity, density, and visual weight. It can also affect SVG stroke-width variation.

Baking

Baking converts the live-subdivided polygon output into a static polygon file stored in polygonSets/. This is useful when you want to use subdivided geometry as input for a further subdivision pass, or when you want to lock in a particular result.

  1. Select a sprite in the upper panel and select a subdivision set in the lower panel.
  2. Preview the result using the eye icon. Confirm the output looks correct.
  3. Click the flame (bake) button in the sets toolbar.
  4. Loom writes the output polygons to polygonSets/, auto-registers the new file, and shows a confirmation with the polygon count.

To export the subdivided geometry as an SVG wireframe without baking, click the folder-arrow (SVG export) button next to the bake button.

Rendering Tab

The Rendering tab defines how sprites are visually drawn. Renderer sets hold one or more renderers; each renderer describes a drawing mode, colours, and optional animation. A sprite is assigned one renderer set and Loom picks from within that set according to the set's playback mode.

Renderer set playback modes

ModeBehaviour
AllDraw every enabled renderer in the set, in list order. This is the normal mode for compositing fill, stroke, brush, and stamp layers together.
SequentialRenderers are applied in order — first renderer for frame 0, second for frame 1, etc.
RandomA renderer is picked at random each frame. Optionally bias towards a named preferred renderer at a specified probability.

Rendering modes

ModeDescription
PointsDraws only the polygon vertices as dots. Controlled by Point Size.
StrokedOutline only, no fill. Controlled by Stroke Colour and Stroke Width.
FilledSolid fill, no outline. Controlled by Fill Colour.
Filled + StrokedBoth fill and outline drawn simultaneously.
BrushedStroke is rendered using a custom brush PNG pattern. Stroke width controls brush scale. Pressure-sensitive geometry can influence opacity and weight.
StenciledA mask image is used to reveal or block the stroke/fill area.
StampedA stamp PNG is tiled or placed along the polygon path with optional meander (random offset) for organic placement.

Brushes and stamps

Brushes and stamps are small PNG images stored in brushes/ and stamps/ respectively. Each renderer in Brushed or Stamped mode holds a list of brush/stamp names with individual enable checkboxes. New projects include default circle, soft-circle, and scatter brushes so brush rendering can be tested immediately.

Creating a brush or stamp

  1. Click Open Brush Editor in the renderer inspector, or use the Brush Editor window directly.
  2. Configure the grid size (rows × columns) and use the drawing tools to paint your pattern.
  3. Use Draw, Erase, Image Draw, or Select / Deselect modes.
  4. Adjust brush size, hardness, and opacity in the tool options panel.
  5. Apply Flip, Rotate, Scale, or Invert transforms as needed.
  6. Click Save (or Save As…) to write the PNG to the project's brushes/ folder. It appears automatically in the brush list.

Meander (Stamped mode)

Meander adds random positional jitter to each stamp placement. Set Meander X and Meander Y for the maximum displacement in pixels. Set a Seed to get a reproducible random sequence.

Progressive brush mode

In Progressive mode, brush strokes can be drawn gradually across frames. With global animation enabled, the path advances from first polygon/edge toward last; in static rendering it draws the complete path.

Colour, width, and opacity drivers

Renderer properties can be static values or animated via drivers. Renderer drivers differ from sprite drivers: they affect only a specific renderer layer. This is especially useful when a renderer set uses Playback All.

PropertyAnimation driver support
Fill colourConstant, Keyframe, Jitter, Noise, Oscillator — see Colour Blend Modes.
Stroke colourConstant, Keyframe, Jitter, Noise, Oscillator — see Colour Blend Modes.
Stroke widthYes — animate width over time using any scalar driver.
Renderer opacityYes — per-renderer alpha multiplier. Fades only this renderer layer, not the whole sprite.
Hold lengthFrames to keep geometry drawn before clearing (ghost/trail effect).

Internal renderer changes

Fill colour, stroke colour, and stroke width changes can operate at different scales. Poly applies changes per subdivision child polygon; Sprite applies one value to the whole sprite; Global applies one value across the render. Random palette changes do not require global animation to be enabled when a static randomised result is desired.

Sprite opacity vs renderer opacity

DriverUse when...
Sprite opacityYou want to fade the whole visible sprite, including every renderer in its assigned renderer set.
Renderer opacityYou want to crossfade or fade only one layer, such as a brush texture over a fill, or a stroke over a stamped base.

Colour blend modes

The Fill Colour Driver and Stroke Colour Driver sections support five modes. Constant and Keyframe work the same way as other drivers. The three new blend modes — Jitter, Noise, and Oscillator — each take two colours (Color A and Color B) and drive a blend value between them every frame.

ModeHow t is computedTypical use
JitterRandom value each frame from a seeded hash. Range (0–0.5) controls the half-width of the window around the midpoint — 0.5 gives full [0,1] swings; 0.1 gives subtle variation.Flickering neon, unstable glow, electric colour
NoiseSmooth continuous random walk. Amplitude controls the peak excursion from the midpoint (0.5 = full range). Period sets the frame interval between noise sample points — higher values give slower, lazier drift.Organic colour breathing, slowly shifting haze, living colour fields
OscillatorPeriodic wave (sine, triangle, square, or sawtooth) normalised to [0,1]. Freq (Hz) sets cycles per second; Phase (0–1) offsets the wave start. Each sprite gets the same phase unless you also vary the sprite's animation seed.Rhythmic pulsing between two colours, colour cycling, strobe fills

Setting up a colour blend driver

  1. In the Rendering tab, select a renderer set and then a renderer within it.
  2. Expand the Fill Colour Driver or Stroke Colour Driver section in the inspector.
  3. Change the Mode picker from Constant to Jitter, Noise, or Oscillator.
  4. Set Color A — this is the base colour the driver starts from (equivalent to the old constant value). Set Color B — the target colour at the other end of the blend.
  5. Adjust the mode-specific parameters: Range / Amplitude / Freq and Phase as needed.
  6. For Jitter and Noise, change Seed to differentiate sprites using the same renderer set — otherwise all sprites will produce identical colour values each frame.
Tip: For a subtle organic breathing effect, use Noise with Amplitude 0.15 and Period 60. For a hard strobe, use Oscillator with a square wave and Freq 2 Hz. For a gentle pulse, use Oscillator with a sine wave and Freq 0.5 Hz.
Jitter and Noise both use the sprite's index as part of their deterministic hash, so different sprites in the same scene will produce different random colour sequences even with the same Seed. The Seed parameter lets you offset those sequences further when you want correlated or uncorrelated groups.

Palette

The inspector offers a Palette section for managing named colour swatches. Palettes can be saved to a JSON file and imported into other projects. This allows colour schemes to be shared across projects without manually re-entering values.

Animation

Loom animates sprite, camera, and renderer properties via drivers — self-contained generators that produce a value for each frame. Sprite drivers affect the entire sprite; renderer drivers affect individual renderer layers inside a renderer set.

Driver types

DriverParametersBehaviour
ConstantValueReturns the same value every frame. Effectively no animation.
JitterAmount, SeedRandom offset each frame from a seeded sequence. Amount controls the ± range.
NoisePeriod, Amplitude, SeedSmooth Perlin noise. Period is the frame-count wavelength; amplitude is the ± range.
OscillatorFrequency, Phase, WaveformSine, triangle, or square wave. Frequency in cycles per second; phase in 0–1 turns.
KeyframeKeyframe list, Loop modeInterpolates between manually placed keyframes. Loop modes: None, Loop, Ping-Pong.

Timeline

The Timeline panel appears at the bottom of the window when a project is open. It shows keyframe lanes for the camera, sprites, and renderer drivers associated with a selected sprite's renderer set. Renderer lanes use a slightly darker background so they are visually distinct from sprite transform lanes.

Working with keyframes

  • Select a sprite and switch to the Keyframe driver in the Animation section of the inspector, or select a renderer driver in the Rendering tab.
  • Click + to add a keyframe at the current playhead frame. The value defaults to the current interpolated property value.
  • Click in the ruler to move the playhead. Drag in the ruler to scrub.
  • Selecting a keyframe also moves the playhead to that keyframe.
  • Drag a keyframe diamond horizontally to change its frame position.
  • Select a keyframe and edit its Frame and Value fields in the inspector. Sprite keyframes switch to the Sprites tab; renderer keyframes switch to the Rendering tab.
  • The matching driver row in the inspector highlights subtly when a keyframe from that lane is selected.
  • Set the Easing per keyframe (Linear, Ease In, Ease Out, Ease In-Out, or a custom curve).
  • Adjust the Loop Mode (None / Loop / Ping-Pong) on the driver to control what happens after the last keyframe.

Renderer keyframe lanes

Renderer lanes are shown beneath the sprite lanes for sprites that use renderer sets with animated renderer drivers. Current renderer lanes are Fill, Stroke, Width, and Opacity. The renderer opacity lane is a per-renderer alpha multiplier and is ideal for crossfading layers in Playback All.

Zoom and scroll

Use the ×1.5 / ÷1.5 buttons to zoom the timeline ruler horizontally. The timeline lane list scrolls vertically when there are more rows than can fit. Drag the larger top grab area to resize the timeline vertically; it can open high enough to inspect dense sprite and renderer lane stacks.

Camera animation

The Timeline has a dedicated Camera section with four keyframe lanes:

LaneEffect
Tracking X / YKeeps the given world-space point centred in the viewport.
Pan X / YTranslates the viewport by the given pixel offsets.
ZoomScales the viewport (1.0 = no zoom; 2.0 = 2× zoom).
RotationRotates the entire canvas around its centre.

Camera animation is evaluated from the same global frame clock as sprite and renderer drivers. It is applied as a final viewport transform after sprite transforms, morph/shape drivers, subdivision, and renderer drivers, so it can be combined with those animation systems.

Depth & Parallax with camera animation

Camera animation becomes a depth-aware parallax system when sprites have non-zero Depth values and Perspective strength is set in the Global tab. The camera lanes in the Timeline remain the same — pan, zoom, tracking, and rotation — but each sprite now evaluates the camera transform through its own depth factor, so every layer moves at a different rate.

Camera laneEffect with depth enabled
Pan X / YThe primary parallax driver. Background sprites (depth > 0) move less; foreground sprites (depth < 0) move more. The focal plane (depth 0) tracks the pan exactly.
ZoomCreates a parallax scale effect. Background layers grow and shrink more slowly than the focal plane during a zoom move, reinforcing depth.
TrackingKeeps the focal-plane point centred. Background sprites will not reach the tracking target exactly — they lag behind proportionally to their depth, which is physically correct.
RotationApplied uniformly across all depths — rotation is not depth-scaled.
Depth and perspective are purely a render-time transform — they do not affect geometry, subdivision, or the geometry editor. The underlying polygon coordinates are always in the flat 2D world.

Morph targets

Morphing smoothly interpolates a sprite's geometry between two states: the base geometry and a morph target geometry. The Morph Amount driver (0 = base, 1 = target) controls the blend.

  1. Create two geometry documents with the same topology (same point count and order).
  2. Assign the base geometry to the sprite as normal.
  3. In the inspector, set the Morph Target field to the second geometry document.
  4. Enable a Morph Amount animation driver (Oscillator or Keyframe for smooth transitions).
  5. Use Morph Lock on the target geometry document to prevent accidental edits.

Export

Loom can export the current rendered output as a PNG still, an SVG vector file, or an MP4/MOV video. All export options are accessible from the toolbar at the top of the window.

Still image

Click the photo icon in the toolbar to save the current frame as a PNG file. The file is written to renders/stills/ with an auto-generated name based on the project name and a timestamp. Resolution is the canvas size multiplied by the Quality setting.

SVG export

Loom has two SVG export paths:

SourceHow to triggerContents
Geometry EditorExport row in the File inspector sectionActive layer's raw geometry as wireframe Bézier paths
Subdivision TabFolder-arrow icon next to the bake buttonSubdivided polygon output as wireframe paths

SVG files are written to the project's svgs/ folder. If a file with the same stem already exists, a numbered suffix is appended automatically. Stroke width varies per path segment when pressure data is available (from freehand drawing).

Video export

  1. Click the movieclapper icon in the toolbar to open the Export Sheet.
  2. Set the desired FPS (24, 25, 30, or 60).
  3. Set the Duration in seconds, or leave at 0 to use the project duration.
  4. Choose a Codec: H.264, HEVC (H.265), or ProRes 4444.
  5. Toggle Restart from beginning if the playhead should reset before export starts.
  6. Click Export…, choose a destination file, and wait for the progress indicator to complete.
The exported video resolution is canvas size × Quality multiplier. Higher quality values increase render time and file size but do not change video duration or frame rate. If a 5-second video takes 30 seconds to render, it still exports as a 5-second video. ProRes 4444 produces larger files but preserves alpha and is better for compositing workflows.

Output folder

Click the folder icon in the toolbar to open the most recently used renders folder in Finder. Loom remembers the last export directory across sessions.
