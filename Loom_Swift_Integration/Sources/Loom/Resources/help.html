What is Loom?

Loom is a generative art environment for creating still images and animations driven by geometric subdivision, procedural rendering, and keyframe animation. It is designed for visual experimentation — each project is a composition of shapes, subdivision rules, and rendering styles that combine to produce imagery ranging from technical line drawings to richly textured organic forms.

Distinguishing features

Subdivision engine
20+ algorithms — Quad, Tri, Split, Echo, Star — each with ratio, inset, and per-wall colour controls.
Animated rendering
Custom brush and stamp PNG patterns, stencil masking, opacity and colour animation on every renderer.
Procedural drivers
Noise, oscillator, jitter, and keyframe drivers can animate any transform or renderer property.
Morph targets
Smoothly interpolate between two geometry states with an animated morph-amount driver.
Sprite hierarchy
Parent–child sprite trees with selectable inherit masks for position, rotation, and scale.
Non-destructive bake
Preview subdivision live, then bake to a permanent polygon file when satisfied.

Core workflow

  1. Create or open a project. The project folder holds all geometry files, brushes, stamps, and the project_config.json.
  2. In the Geometry tab, draw or import polygon/curve shapes using the built-in editor.
  3. In the Sprites tab, create sprite instances and assign your geometry. Set position, scale, and rotation. Establish parent–child hierarchies if needed.
  4. In the Subdivision tab, define sets of subdivision parameters and assign them to sprites. Preview the subdivided result and bake when ready.
  5. In the Rendering tab, create renderer sets — choose fill/stroke modes or brush/stamp patterns. Assign each renderer set to the sprites that should use it.
  6. Add animation via the Sprites inspector or the Timeline. Use drivers (noise, oscillator, keyframe) for each property you want to animate.
  7. Press Play (Space) to preview. Export as a still, SVG, or video from the toolbar.

Project structure

Every Loom project is a folder inside ~/.loom_projects/ (default) with the following layout:

Quick Start

New project

  1. Choose File → New Project… (or ⌘N). Enter a name in the dialog and click Create.
  2. Loom creates the project folder and opens project_config.json with sensible defaults (800×800 canvas, 30 fps).
  3. The window switches to the main layout — the five tab panels and playback toolbar become active.

Open an existing project

  1. Choose File → Open Project… (or ⌘O) and select the project folder, or click the Open Loom Projects Folder icon in the toolbar and double-click a project.
  2. Loom reads project_config.json and reloads all geometry, renderers, and animation state.
  3. Use ⌘R or the reload button to re-read the config after making external edits.

Geometry Tab

The Geometry tab manages the polygon and curve files that define all shapes in your project. A geometry document holds one or more named layers, each containing polygons, curves, and points.

Geometry list

The left panel shows all registered geometry sources. Each row displays the file name and type indicators (polygon, curve, point icons) alongside a sprite-count badge. Click to select; double-click to rename. Right-click for Rename, Duplicate, and Delete options.

Tip: Geometry files are shared — multiple sprites can reference the same file with different subdivision sets and renderers applied.

Canvas navigation

  • Use the +/− zoom buttons to double or halve the view scale.
  • Enable the Grid at three detail levels: Quadrants, Standard, and Fine.
  • Toggle Control Points to show or hide Bézier handles.
  • Use the Centre button to re-center the view on the geometry.

Selection and editing

  • Click to select individual points or segments; drag a region to multi-select.
  • Anchor-only mode hides control handles so you edit only anchor positions.
  • Auto-weld merges nearby points automatically (configurable tolerance).
  • Undo / Redo via the toolbar buttons or standard keyboard shortcuts.
  • Copy / Paste duplicates geometry relative to centroid position.

Layers

Each geometry document supports multiple named layers. Use layers to separate shape groups — for example, keeping outline polygons on one layer and detail curves on another.

  • Click a layer to make it active (new geometry is drawn on the active layer).
  • Toggle the eye icon to show/hide a layer; toggle the pencil icon to lock/unlock editing.
  • Double-click a layer name to rename it.
  • Add or remove layers with the + / − buttons in the layers panel.

Sprites Tab

Sprites are the fundamental render units in Loom. Each sprite references a geometry file and is assigned a subdivision set and a renderer set. The Sprites tab shows a hierarchical tree of all sprites organized into named sets.

Sprite list

Sprites are grouped into sprite sets. Click the chevron to collapse or expand a set. Drag sprites within the list to reorder them, or drag between sets to reassign. Use the search bar at the top to filter by sprite or set name.

Each sprite row shows:

  • An enable checkbox — disabled sprites are not drawn.
  • The sprite name (double-click to rename).
  • A play icon if any animation driver is active.
  • For child sprites, P / R / S buttons to toggle parent transform inheritance.

Creating sprites

  1. Click + in the set toolbar to add a new sprite set, or use the sprite + button to add a sprite to the selected set.
  2. With the new sprite selected, open the inspector and assign a Geometry (polygon set file), a Renderer Set, and a Subdivision Set.
  3. Set initial Position, Scale, and Rotation in the Transform section.

Hierarchy and inheritance

Any sprite can be assigned a parent via the Hierarchy section in the inspector. Child transforms are applied relative to the parent. Use the P / R / S toggles to choose which parent transforms the child inherits:

ToggleEffect when on
P (Position)Child offset is applied relative to the parent's world position.
R (Rotation)Child rotation is added to the parent's world rotation.
S (Scale)Child scale is multiplied by the parent's world scale.
Tip: Set all three off to create a sprite that uses the parent only for logical grouping, with completely independent world-space transforms.

Sprite inspector sections

SectionContents
SpriteName, renderer set, subdivision set
TransformX/Y position, X/Y scale, rotation in degrees
AnimationPer-property animation drivers (see Animation)

Subdivision Tab

The Subdivision tab is where Loom's generative power lives. It manages subdivision parameter sets — ordered sequences of algorithms that progressively fracture geometry into more complex polygonal structures.

Working with sets

  1. Click + in the Sets toolbar to create a new subdivision set and give it a name.
  2. Click + in the Parameters toolbar to add the first algorithm to the set.
  3. Select the parameter row and adjust its settings in the inspector on the right: algorithm type, ratios, inset, visibility rules, and optional PTW/PTP geometric transforms.
  4. Click the eye icon next to the set to preview the subdivided result on the canvas — the sprites assigned to this set update live.
  5. When satisfied, click Apply in the Apply Bar to commit the set assignment to the selected sprite, or use the bake button to write the subdivided geometry to a permanent file.

PTW — Polygons Transform Whole

PTW applies geometric transformations to entire output polygons as units. When enabled, each polygon produced by the subdivision step can be independently translated, scaled, and rotated by a randomly sampled amount within configurable ranges. This breaks the regularity of subdivided geometry, producing scattered, fragmented, or organically varied results from otherwise structured forms.

ParameterDescription
EnabledMaster on/off toggle for the entire PTW block.
WholeWhen on, the transform is applied to the whole polygon shape as a rigid body. When off, only positional displacement is applied.
ProbabilityPercentage chance (0–100) that the transform fires for each individual polygon. At 50%, roughly half the output polygons will be affected each frame.
Common centreWhen on, all transforms use a single shared centre point rather than each polygon's own centroid. Useful for creating radial scatter effects.
Ran translEnable random translation. When off, no positional displacement is applied even if translation ranges are set.
Ran scaleEnable random scaling.
Ran rotateEnable random rotation.
Transl X (mn/mx)Min/max range for random displacement along the X axis.
Transl Y (mn/mx)Min/max range for random displacement along the Y axis.
Scale X (mn/mx)Min/max range for random X-axis scale factor.
Scale Y (mn/mx)Min/max range for random Y-axis scale factor.
Rot range (mn/mx)Min/max range for random rotation angle.
Tip: Set Probability below 100 and Whole off with a small Transl range to add subtle positional noise to subdivided shapes without destroying their overall structure.

PTP — Polygons Transform Points

PTP operates at the individual point level inside each output polygon. Rather than moving whole shapes, it deforms specific sets of Bézier anchor points and control handles — creating spikes, tears, curvature changes, and organic distortions. PTP is organised into five independent groups accessed via tabs, each targeting a different class of points within the polygon.

All five groups share a top-level Enabled toggle and Probability (applied per polygon). Each group has its own secondary probability.

Ext — Exterior Anchors

Displaces the outer anchor points of each polygon outward or inward — creating spikes, dents, or irregular protrusions. The target set can be all anchors, corners only, or midpoint anchors. Spike direction can be symmetrical, left-biased, right-biased, or random. Control point handles can optionally follow the displaced anchor with a configurable multiplier, and a squeeze option pulls handles toward the edge midpoint for sharper transitions.

ParameterDescription
ProbabilityChance this group fires per polygon.
Spike factorDisplacement magnitude. Positive values push outward; negative pull inward.
Which spikeAll — every anchor; Corners — corner anchors only; Middles — midpoint anchors only.
Spike typeSymmetrical — balanced spike; Right / Left — asymmetric lean; Random — varies per anchor.
Spike axisConstrain displacement to XY (both), X, or Y only.
Ran spike / Spike rangeRandomise spike magnitude within min/max range.
CPs follow / CPs multControl handles track the displaced anchor. Multiplier scales how closely they follow.
Ran CPs follow / CPs follow rangeRandomise the CP follow amount per anchor.
CPs squeeze / Squeeze factorPull control handles toward the edge midpoint, sharpening the spike base.
Ran squeeze / Squeeze rangeRandomise squeeze amount per anchor.

Cen — Central Anchors

Tears or displaces anchor points positioned near the polygon interior, pulling them in a specified direction and axis. An all points follow option propagates the tear displacement to the rest of the polygon's points, effectively shearing the whole shape. Inverted follow reverses this propagation direction.

ParameterDescription
ProbabilityChance this group fires per polygon.
Tear factorDisplacement magnitude of the tear.
Tear axisConstrain tear to XY, X, Y, or Random.
Tear directionDiagonal / Left / Right / Random — directional bias of the tear.
Ran tear / Tear rangeRandomise tear magnitude within min/max range.
CPs follow / CPs multControl handles follow the torn anchor.
Ran CPs follow / CPs follow rangeRandomise CP follow amount.
All pts followAll other polygon points also displace in the same direction as the tear.
Inverted followReverse the direction of the all-points-follow displacement.

OCP — Outer Control Points

Repositions the Bézier control handles on the exterior edges of the polygon to introduce curvature. Handles can be oriented perpendicular to the edge, or directed from the polygon centre outward. The curve type controls whether individual edges bow outward (puff) or inward (pinch), and how these alternate around the polygon perimeter.

ParameterDescription
ProbabilityChance this group fires per polygon.
Line ratio X / YBase position of the control handle along the edge (0 = start anchor, 1 = end anchor).
Ran ratio / Inner range / Outer rangeRandomise ratio position within inner and outer min/max ranges.
Curve modePerpendicular — handles project at 90° to the edge; From centre — handles point away from the polygon centroid.
Curve typePuff — all edges bow outward; Pinch — all edges bow inward; alternating combinations (Puff-Pinch-Puff-Pinch, etc.) create star or flower-like shapes.
Mult min / maxCurvature multiplier range — higher values produce more extreme bowing.
Ran mult / Mult rangeRandomise the multiplier per edge within min/max range.
From ctr X / YFor From centre mode: ratio offsets that control how far from the centroid the handle is placed.
Ran from ctr / From ctr A / B rangesRandomise the from-centre position within two independent ranges.

ALC — Anchors Linked to Centre

Tears anchor points that are geometrically linked to the polygon centre — pulling them toward an outside corner, the opposite corner, toward the centre itself, or randomly. Similar to Cen but directional tear types reference polygon structure (corners and edges) rather than a global axis.

ParameterDescription
ProbabilityChance this group fires per polygon.
Tear factorDisplacement magnitude.
Tear typeOutside corner — pulls toward the nearest exterior corner; Opposite corner — pulls toward the far corner; Towards centre — pulls inward; Random — varies per anchor.
Ran tear / Tear rangeRandomise tear magnitude within min/max range.
CPs follow / CPs multControl handles track the torn anchor.
Ran CPs follow / CPs follow rangeRandomise CP follow amount.

ICP — Inner Control Points

Controls the inward-facing Bézier handles on polygon edges. These can operate independently or reference the behaviour of the outer control points (OCP). The refer outer modes let inner handles mirror, exaggerate, or oppose what OCP has done, creating coordinated curvature effects from a single set of parameters. Common line aligns handles across even, odd, or randomly chosen edges.

ParameterDescription
ProbabilityChance this group fires per polygon.
Refer outerNone — independent; Follow — mirrors OCP offset; Exaggerate — amplifies OCP offset; Counter — opposes OCP offset.
Inner mult X / YMultiplier applied to the inward handle position on X and Y axes.
Outer mult X / YMultiplier applied to the outward handle position on X and Y axes.
Inner ratio / Outer ratioBase position ratios for inner and outer handles along the edge.
Ran ratio / Inner range / Outer rangeRandomise ratios within independent inner and outer min/max ranges.
Common lineEven / Odd — align handles on even or odd-indexed edges; Random — random per edge; None — no alignment.

Baking

Baking converts the live-subdivided polygon output into a static polygon file stored in polygonSets/. This is useful when you want to use subdivided geometry as input for a further subdivision pass, or when you want to lock in a particular result.

  1. Select a sprite in the upper panel and select a subdivision set in the lower panel.
  2. Preview the result using the eye icon. Confirm the output looks correct.
  3. Click the flame (bake) button in the sets toolbar.
  4. Loom writes the output polygons to polygonSets/, auto-registers the new file, and shows a confirmation with the polygon count.

To export the subdivided geometry as an SVG wireframe without baking, click the folder-arrow (SVG export) button next to the bake button.

Rendering Tab

The Rendering tab defines how sprites are visually drawn. Renderer sets hold one or more renderers; each renderer describes a drawing mode, colours, and optional animation. A sprite is assigned one renderer set and Loom picks from within that set according to the set's playback mode.

Renderer set playback modes

ModeBehaviour
SequentialRenderers are applied in order — first renderer for frame 0, second for frame 1, etc.
RandomA renderer is picked at random each frame. Optionally bias towards a named preferred renderer at a specified probability.

Rendering modes

ModeDescription
PointsDraws only the polygon vertices as dots. Controlled by Point Size.
StrokedOutline only, no fill. Controlled by Stroke Colour and Stroke Width.
FilledSolid fill, no outline. Controlled by Fill Colour.
Filled + StrokedBoth fill and outline drawn simultaneously.
BrushedStroke is rendered using a custom brush PNG pattern. Stroke width controls brush scale.
StenciledA mask image is used to reveal or block the stroke/fill area.
StampedA stamp PNG is tiled or placed along the polygon path with optional meander (random offset) for organic placement.

Brushes and stamps

Brushes and stamps are small PNG images stored in brushes/ and stamps/ respectively. Each renderer in Brushed or Stamped mode holds a list of brush/stamp names with individual enable checkboxes.

Creating a brush or stamp

  1. Click Open Brush Editor in the renderer inspector, or use the Brush Editor window directly.
  2. Configure the grid size (rows × columns) and use the drawing tools to paint your pattern.
  3. Use Draw, Erase, Image Draw, or Select / Deselect modes.
  4. Adjust brush size, hardness, and opacity in the tool options panel.
  5. Apply Flip, Rotate, Scale, or Invert transforms as needed.
  6. Click Save (or Save As…) to write the PNG to the project's brushes/ folder. It appears automatically in the brush list.

Meander (Stamped mode)

Meander adds random positional jitter to each stamp placement. Set Meander X and Meander Y for the maximum displacement in pixels. Set a Seed to get a reproducible random sequence.

Colour and animation

Renderer colours can be static RGBA values or animated via drivers:

PropertyAnimation driver support
OpacityYes — fade in/out, oscillate, keyframe.
Fill colourYes — animate RGBA over time.
Stroke colourYes — animate RGBA over time.
Stroke widthYes — animate width over time (change amount driver).
Hold lengthFrames to keep geometry drawn before clearing (ghost/trail effect).

Palette

The inspector offers a Palette section for managing named colour swatches. Palettes can be saved to a JSON file and imported into other projects. This allows colour schemes to be shared across projects without manually re-entering values.

Animation

Loom animates sprite and renderer properties via drivers — self-contained generators that produce a value for each frame. Every animatable property (position, scale, rotation, opacity, morph amount, shape index) has its own driver.

Driver types

DriverParametersBehaviour
ConstantValueReturns the same value every frame. Effectively no animation.
JitterAmount, SeedRandom offset each frame from a seeded sequence. Amount controls the ± range.
NoisePeriod, Amplitude, SeedSmooth Perlin noise. Period is the frame-count wavelength; amplitude is the ± range.
OscillatorFrequency, Phase, WaveformSine, triangle, or square wave. Frequency in cycles per second; phase in 0–1 turns.
KeyframeKeyframe list, Loop modeInterpolates between manually placed keyframes. Loop modes: None, Loop, Ping-Pong.

Timeline

The Timeline panel appears at the bottom of the window when a project is open. It shows keyframe lanes for every animated sprite property.

Working with keyframes

  • Select a sprite and switch to the Keyframe driver in the Animation section of the inspector.
  • Click + to add a keyframe at the current frame. The value defaults to the current property value.
  • Drag a keyframe diamond horizontally to change its frame position.
  • Select a keyframe and edit its Frame and Value fields in the inspector.
  • Set the Easing per keyframe (Linear, Ease In, Ease Out, Ease In-Out, or a custom curve).
  • Adjust the Loop Mode (None / Loop / Ping-Pong) on the driver to control what happens after the last keyframe.

Zoom and scroll

Use the ×1.5 / ÷1.5 buttons to zoom the timeline ruler horizontally. Drag the timeline panel's top edge to resize it vertically (range 80–320 px).

Camera animation

The Timeline has a dedicated Camera section with three keyframe lanes:

LaneEffect
Pan X / YTranslates the viewport by the given pixel offsets.
ZoomScales the viewport (1.0 = no zoom; 2.0 = 2× zoom).
RotationRotates the entire canvas around its centre.

Morph targets

Morphing smoothly interpolates a sprite's geometry between two states: the base geometry and a morph target geometry. The Morph Amount driver (0 = base, 1 = target) controls the blend.

  1. Create two geometry documents with the same topology (same point count and order).
  2. Assign the base geometry to the sprite as normal.
  3. In the inspector, set the Morph Target field to the second geometry document.
  4. Enable a Morph Amount animation driver (Oscillator or Keyframe for smooth transitions).
  5. Use Morph Lock on the target geometry document to prevent accidental edits.

Export

Loom can export the current rendered output as a PNG still, an SVG vector file, or an MP4/MOV video. All export options are accessible from the toolbar at the top of the window.

Still image

Click the photo icon in the toolbar to save the current frame as a PNG file. The file is written to renders/stills/ with an auto-generated name based on the project name and a timestamp. Resolution is the canvas size multiplied by the Quality setting.

SVG export

Loom has two SVG export paths:

SourceHow to triggerContents
Geometry EditorExport row in the File inspector sectionActive layer's raw geometry as wireframe Bézier paths
Subdivision TabFolder-arrow icon next to the bake buttonSubdivided polygon output as wireframe paths

SVG files are written to the project's svgs/ folder. If a file with the same stem already exists, a numbered suffix is appended automatically. Stroke width varies per path segment when pressure data is available (from freehand drawing).

Video export

  1. Click the movieclapper icon in the toolbar to open the Export Sheet.
  2. Set the desired FPS (24, 25, 30, or 60).
  3. Set the Duration in seconds, or leave at 0 to use the project duration.
  4. Choose a Codec: H.264, HEVC (H.265), or ProRes 4444.
  5. Toggle Restart from beginning if the playhead should reset before export starts.
  6. Click Export…, choose a destination file, and wait for the progress indicator to complete.
The exported video resolution is canvas size × Quality multiplier. ProRes 4444 produces larger files but preserves alpha and is better for compositing workflows.

Output folder

Click the folder icon in the toolbar to open the most recently used renders folder in Finder. Loom remembers the last export directory across sessions.
