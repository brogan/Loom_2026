Reference Image

  1. Click Browse in the Reference Image panel at the top.
  2. Select a PNG, JPEG, or GIF file. The image is displayed behind the polygon geometry.
  3. Toggle visibility with the Toggle Reference Image toolbar button.
The reference image is a session-only drawing aid and is not saved with the polygon set.

Saving

Name Field

Type a name in the Name field before saving. Spaces become underscores; non-alphanumeric characters are stripped; the result is lowercased. This becomes the base filename for all exported files.

Open curves and point sets are written automatically alongside the polygon save if they are present on the canvas. You do not need separate save steps.

If both closed polygons and open curves exist on the same canvas, they are saved to separate files in different directories. The Loom Parameter Editor expects them to be registered separately in the Polygons and Curves sub-tabs respectively. For clarity, keep closed polygon sessions and open curve sessions separate.

Layer Bundles

When the canvas has polygons on two or more layers, Save writes:

  • One {name}_{layerName}.xml per layer containing polygons.
  • A {name}.layers.xml manifest file listing the layer names and their corresponding filenames.

The Loom Parameter Editor detects the .layers.xml manifest and imports the whole bundle as a layered polygon set. Each layer can then be given a different renderer in Loom.

SVG Export & Import

Automatic SVG Export

Every time you click Save, an SVG copy is written alongside the XML in the svgs/ directory. Each closed polygon becomes a <path> element using M and C commands in a viewBox="0 0 1000 1000" SVG. Open curves are exported without a closing Z command.

SVG Import

  1. Click Import SVG in the SVG panel (top area).
  2. Select any SVG file. The browser opens in the svgs/ directory by default.
  3. Each <path> in the SVG is added to the canvas as new polygon geometry. Existing geometry is preserved.

Supported SVG Path Commands

CommandHandling
M / mMove to (subpath start).
C / cCubic Bézier — used directly.
L / lLine to — converted to cubic with control points at ⅓ and ⅔.
Q / qQuadratic Bézier — converted to cubic via degree elevation.
Z / zClose path.
H, V, S, T, A, …Silently skipped.

Coordinates are mapped from the SVG viewBox into Bezier canvas space automatically. SVG Import is undoable.

Undo

Press ⌘ Cmd + Z to undo the last operation. Up to 20 undo levels are kept. Undo restores complete polygon geometry, all weld links, and layer assignments. The current selection is cleared after an undo.

There is no Redo. If you undo too far, you will need to redo the work manually.

Operations that create undo points

  • Any drag start (point, edge, or polygon)
  • First movement of a Scale or Rotate slider gesture
  • Close Polygon
  • Finish Open Curve
  • Place discrete point (Create Points mode — one snapshot per click)
  • Clear All Geometry
  • Zoom In / Zoom Out (each click)
  • Snap Anchors / Snap Anchors & Controls
  • Flip Horizontal / Flip Vertical / Centre
  • Weld (explicit and drag-to-weld)
  • Weld All Adjacent
  • Extrude (start of drag)
  • Knife cut (on release)
  • Intersect (quad mesh)
  • Duplicate
  • Delete Selected
  • SVG Import
  • Paste (⌘ Cmd + V)
